# 沉浸音频、歌词与 K 歌功能迁移改造方案

## 一、目标

以当前 `main` 为唯一代码基线，选择性迁移 `feature/audio-lyrics-karaoke-20260629` 中的纯音频播放、歌词和 K 歌能力，避免旧分支覆盖当前播放器、MPV、ISO、LUT、手势、弹幕、播放性能、备份恢复及 UI 修复。

新增一个设置开关：`沉浸音频模式`。

- 默认关闭：完全使用当前原生播放界面，不初始化歌词、音频队列、K 歌、录音、音高检测和评分模块。
- 开启后：仅在识别为音频型内容时进入沉浸音频界面，并启用歌词、音频队列、K 歌等专属能力。
- 普通视频始终使用原生界面。
- 播放中切换开关时尽量复用当前播放器、媒体地址、播放进度、暂停状态和倍速，不重新解析资源。
- 原生界面与沉浸音频界面功能隔离，不在原生界面插入歌词或 K 歌入口。

## 二、迁移原则

1. 不直接 merge 旧功能分支，不使用整文件 `ours/theirs` 覆盖。
2. 新增的独立类、模型、图标和模型文件可选择性复制；已有文件按当前 main 逐处接线。
3. 当前 main 的三播放器体系、手动切换策略、播放性能参数、字幕、弹幕、LUT、ISO、网盘播放和手势逻辑优先级最高。
4. 沉浸音频逻辑必须受统一开关控制；关闭时不产生额外网络、线程、录音和推理开销。
5. 手机端和电视端分别适配，不能用一端布局覆盖另一端。
6. 所有新增弹窗和按钮遵循当前蓝紫色 UI、紧凑按钮、小圆角矩形和焦点态规范。
7. 每一阶段独立编译、提交和验证，便于回滚与定位回归。

## 三、功能范围

### 3.1 本次核心迁移

- 沉浸音频播放界面。
- 音频封面、背景、播放状态和进度展示。
- 音频播放队列及本地持久化。
- 多来源歌词搜索、匹配、解析和逐字歌词。
- 歌词手动搜索、选择、重载和进度同步。
- K 歌音轨获取、导入和生成。
- 麦克风录音、音高检测、实时评分和结算页。
- UltraStar、MIDI 等 K 歌轨道解析。
- Basic Pitch TFLite 模型与生成流程。
- 手机端和电视端对应 UI。

### 3.2 明确排除

- 旧分支的播放器内核、播放器切换和解码逻辑。
- 旧分支对 MPV、FFmpeg、IJK、LUT、ISO 和播放性能代码的删除或回退。
- Updater、Git 云盘、备份、CSP 等与沉浸音频无关的旧实现。
- 旧分支中与当前 main UI 规范冲突的公共弹窗样式。

### 3.3 独立迁移项

旧分支还补充了电视端的 DLNA、扫码和睡眠计时器。当前 main 手机端已有对应功能，电视端缺少。它们不属于沉浸音频核心，放在独立阶段和独立提交中评估、迁移和测试。

## 四、架构设计

### 4.1 设置开关

新增统一设置：

```text
沉浸音频模式：关闭 / 开启
```

建议由 `PlayerSetting` 保存，手机和电视设置页读取同一配置。

### 4.2 模式判定

进入沉浸音频界面必须同时满足：

```text
沉浸音频模式已开启 && 当前内容被识别为音频型内容
```

音频型内容判定优先使用明确 MIME/format、音频轨道信息和站点返回类型；扩展名只作为兜底。不能仅凭无视频画面立即切换，避免网络加载阶段误判。

### 4.3 生命周期

- 进入沉浸模式时创建歌词和 K 歌控制器。
- 离开沉浸模式、切集、退出播放或 Activity 销毁时释放搜索任务、歌词循环、录音、音高检测、模型推理和临时文件。
- 切换界面不重建 `PlayerManager`。
- 关闭开关后立即退出沉浸 UI，并停止其专属后台任务。

### 4.4 UI 隔离

- 原生播放布局保持当前 main 结构。
- 沉浸音频布局作为独立容器叠加或切换显示。
- 不删除原生控制栏，不用沉浸布局替换根布局。
- 两套 UI 只共享播放器状态，不共享专属按钮和业务入口。

## 五、冲突处理策略

| 冲突区域 | 处理方式 |
| --- | --- |
| `PlayerManager` | 保留 main，全量审查后仅增加音频模式需要的状态接口 |
| `VideoActivity` | 以 main 为底，逐块接入沉浸音频生命周期与 UI |
| `PlaybackActivity` | 保留 main 的播放器服务和旋转逻辑，只增加必要回调 |
| `PlayerSetting` | 保留全部现有设置，仅新增沉浸音频开关和音频专属配置 |
| `ExoUtil` | 保留当前播放器优化，只补充 K 歌音频提取确实需要的能力 |
| 公共弹窗/选择器 | 使用 main 当前样式，不覆盖公共资源 |
| 字符串 | 按 key 合并，禁止整文件覆盖 |
| 手机/电视布局 | 分别迁移并验证，不相互复制覆盖 |

## 七、验收标准

1. 开关关闭时，原生界面、功能和性能与当前 main 一致。
2. 开关开启且播放音频时，稳定进入沉浸音频界面。
3. 歌词和 K 歌只在沉浸音频模式中存在和运行。
4. UI 切换不重新解析媒体、不重置进度、不自动切换播放器。
5. 三种播放器均不因新功能崩溃；能力差异应给出明确提示。
6. 退出或关闭模式后，麦克风、线程、网络任务和模型推理均释放。
7. 当前 main 的视频播放、网盘、ISO、LUT、字幕、弹幕和手势无回归。
